Projects

Banking System


Motivation of the Project

It is a simple but useful system that is crucial in today's world. Personally, it helps improve my knowledge of java and is a way for me to bring everything I have learnt about coding together.

What the Project does

br> It implements a simple banking system that involves the elements - the bank itself, the accounts in the bank and the customers who have the accounts in the bank. It involves the different actions such as deposit, withdraw, transfer and get balance and so on. Accounts and corresponding customers can be added and removed. It covers a majority of the operations that make up a banking system in the real world.

Project Design

5 classes were created - Bank, BankCustomer, BankAccount, CheckingAccount and SavingsAccount The BankAccount class was an abstract class, which was extending the CheckingAccount and SavingsAccount classes These are the main elements of the banking system so it is important that each of them have their own class 2 interfaces were created - Account and Customer The BankAccount abstract class implements the Account interface and the BankCustomer class implements the Customer interface These interfaces provide the framework for the classes. Test cases for all the methods were crated. Test-driven implementation was used. The test cases were written out first before the classes were implemented. This design documents what each feature is going to do, focuses attention on boundary cases and ensures that the code is testable. The whole program was coded in java.

Implementation

I was responsible for implementing the Account interface, the BankAccount abstract class, and the CheckingAccount and SavingsAccount classes. I wrote out the test cases for these as well. With the CheckingAccount and SavingsAccount classes, I was also able to add some complexity to our banking system by involving elements like minimum balance, overdraft and penalty fees, withdrawal limits and so on.
